# 地图实验页

## 目的

当前 `APP-PLUS` 下，情报 tab 首屏直接挂载地图 `web-view` 会触发原生崩溃。  
为了继续定位问题，同时不影响首页稳定性，新增了独立实验页：

- 路由：`/pages/map/lab`
- 页面文件：`pages/map/lab.vue`

## 验证模式

实验页提供三种模式，按风险从低到高逐步验证：

1. `本地空白页`
   - 地址：`/static/map/lab-blank.html`
   - 只验证最轻量 `web-view` 是否稳定

2. `本地桥接页`
   - 地址：`/static/map/fuyaomap-bridge.html?debugFallback=1&showHud=1`
   - 验证本地静态 bridge 壳本身是否稳定

3. `远程轻量页`
   - 地址：`http://159.75.54.99:8002/embed/map?readonly=1&debug=1&lite=1`
   - 远程页面可加载，但跳过地图内核初始化

4. `远程地图内核`
   - 地址：`http://159.75.54.99:8002/embed/map?readonly=1&debug=1&kernel=1`
   - 直接验证线上嵌入地图页的地图内核是否会触发问题

## 使用方式

1. 打开 App，进入“情报”页
2. 点击右上角的“地图实验页”入口
3. 先选 `本地空白页`，点击“加载实验”
4. 若稳定，再测试 `本地桥接页`
5. 再测试 `远程轻量页`
6. 最后测试 `远程地图内核`

每次只验证一种模式。  
如果某一种模式一加载就崩，说明问题已经收敛到这一层承载链路。

当前已验证结论：

- 本地空白页：稳定
- 本地桥接页：稳定
- 远程轻量页：用于验证远程页面承载
- 远程地图内核：当前 Android App 宿主中会触发原生崩溃

## 当前意义

实验页用于排查原生崩溃根因，不替代首页地图承载。  
情报首页目前仍保持安全模式，以保证 App 正常可用。
